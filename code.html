Encoding An Experience

Component Type Number of Bits/Range Description
Weather Binary 2 bits(4 different values) The current weather of the tournament venue. An unknown value is marked as 00. A rainy value is marked as 01. A sunny value is marked as 10. A snowy value is values as 11.,This encoding system ensures that unnecessary bytes aren’t used to encode text.
Description of Venue ASCII 4000 bits(0 - 500 chars) Describing the overall structure of the venue is an important part of recreating the experience. Since one ascii character takes up 8 bits (or 1 byte), 4,000 bits allows us to send a complete paragraph about the visual aspect of the experience. It is unreasonable to receive a complete essay describing the appearance of the venue, so I do not need to offer more than 500 characters.
Occupancy ASCII 5 bits( 32 values) Number is the total amount of people within the venue. 5 bits offers 32 total values of people participating inside the tournament. The venue I go to typically reachers 16 people, but the extra space ensures that the experience can be recreated without the risk of an overflow.
Coordinates Number 2 separate 6 bits (128 values but only 92 are needed for each state.) Longitude and latitude both reach a cap of 90. However, that value can either be negative or positive, so it is important to have two extra values that describe the sign of the coordinates. These specific values take up much less space than the name of the street that the venue is on.
Hours Open Number 2 Separate 5 bits(32 values, but only 24 are needed for each state) First value is the time the venue opens while the second value is the time the venue closes. These values are both in military time to save space while encoding the data. In addition, this format assumes that the tournament opens at a whole number which is a safe assumption to make. Typically, the first hour of a tournament is dedicated to friendly matches, so anyone receiving this message will not miss out on the actual event.

Project Explanation

This project required students to recreate a significant event through the use of ASCII and binary code. The event was broken down to its simplest elements to easily transmit the information to another student. I was expected to justify the encoding systems used to further my understanding of this code.

Project Reflection

I learned more about the specific uses of Binary and ASCII code to transmit information. Consistently minimizing the amount of data sent to just the bare essentials resulted in a challenging final product. I wanted to ensure that every single step accurately represented an efficient compression system.
